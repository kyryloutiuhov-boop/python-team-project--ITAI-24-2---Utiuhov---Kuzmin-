Генератор паролів (Командний проєкт)

Опис проєкту

Даний проєкт є консольним генератором паролів, розробленим у рамках командної роботи.
Програма дозволяє користувачу згенерувати пароль заданої довжини, обрати використання цифр та спеціальних символів, а також оцінити рівень надійності пароля.

Структура проєкту

         project/
         │
         ├── src/
         │   ├── main.py              # інтерфейс користувача та точка входу
         │   └── password_utils.py    # логіка генерації пароля та оцінка надійності
         │
         │── tests/
         │   ├── test_password.py     # файл для тестування функцій
         │
         ├── README.md

Розподіл відповідальності

Студент A (Team Lead / Інтерфейс):

 реалізував інтерфейс користувача у файлі `main.py`
 забезпечив введення даних (довжина пароля, параметри)
 виведення результатів
 інтеграцію всіх частин проєкту

Студент B (Developer / Логіка):

 реалізував функцію генерації пароля у `password_utils.py`
 реалізував перевірку надійності пароля
 виконав тестування функціоналу

Як запустити програму

1. Відкрити термінал у папці проєкту
2. Перейти в папку `src`:

         cd src

3. Запустити програму:

         python main.py

Тестування

Для перевірки коректності роботи функцій створено окремий файл тестів:

         tests/test_password.py

Запуск тестів:

         python tests/test_password.py

Тестування перевіряє:

правильність довжини згенерованого пароля
наявність або відсутність цифр
коректність визначення рівня надійності

Функціональні можливості

 генерація пароля заданої довжини
 можливість додавання цифр
 можливість додавання спеціальних символів
 оцінка надійності пароля (Слабкий / Середній / Сильний)

Організація роботи з Git

 кожен учасник працював у власній гілці
 зміни вносилися через Pull Request
 проводилась взаємна перевірка коду (code review)
 після перевірки зміни об’єднувались у гілку `main`

Висновок

У процесі виконання проєкту було отримано практичні навички командної розробки програмного забезпечення з використанням Git та GitHub.
Було освоєно розподіл задач, роботу з гілками, створення Pull Request, проведення code review та інтеграцію коду у спільний проєкт.
